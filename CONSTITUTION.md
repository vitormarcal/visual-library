# CONSTITUTION.md

## Mission

This project exists to provide a personal visual library for curated internet images.

It is not a photo backup system.
It is not a social network.
It is not a cloud storage platform.
It is not an AI-first image analysis product.

The product is a low-friction visual bookmarking and rediscovery tool.

Users should be able to:

- quickly save images they find online;
- organize them visually;
- rediscover them later through browsing, tags, collections, and visual grouping;
- build personal aesthetic/reference libraries.

The emotional model is closer to:

- a personal Pinterest;
- a visual notebook;
- a moodboard archive;
- a curated inspiration vault.

Not:
- Google Photos;
- Immich;
- Dropbox;
- Instagram.


---

# Product Philosophy

## Simplicity over completeness

The product should remain intentionally narrow.

Every feature must justify its existence against:
- complexity cost;
- UI pollution;
- cognitive overhead;
- maintenance burden.

The default answer to new features should be:
"Do we really need this?"

Avoid enterprise thinking.

Avoid platform thinking.

Avoid solving hypothetical future scale problems.


---

# Primary User Behaviors

The system is optimized for:

- saving interesting images from the internet;
- collecting wallpapers;
- collecting visual references;
- organizing aesthetic inspirations;
- browsing visually;
- rediscovering forgotten images.

The system is NOT optimized for:

- family albums;
- camera roll replacement;
- timeline memories;
- professional DAM workflows;
- collaborative media management;
- corporate asset governance.


---

# UX Principles

## Low friction is mandatory

Saving and organizing images must feel immediate.

The product should minimize:
- clicks;
- forms;
- modal overload;
- confirmation dialogs;
- setup complexity.

The system should encourage flow-state browsing.

## Visual-first experience

Images are the interface.

Metadata exists to support rediscovery, not to dominate the screen.

Prioritize:
- masonry layouts;
- visual density;
- fast browsing;
- smooth scrolling;
- large thumbnails;
- minimal chrome.

Avoid:
- dashboard aesthetics;
- admin-panel feeling;
- table-heavy interfaces.

## Collections and tags have distinct purposes

Tags:
- lightweight;
- flexible;
- associative.

Collections:
- curated;
- intentional;
- human-organized.

Do not collapse both concepts into a single abstraction.


---

# Scope Discipline

This project must resist feature creep.

Avoid turning the system into:

- a cloud sync platform;
- a social media product;
- an AI image generator;
- an image editor;
- a productivity suite;
- a note-taking app;
- a generic CMS.

New capabilities must reinforce the core mission:
personal visual curation and rediscovery.


---

# AI Guidance

AI agents contributing to this project must:

- preserve simplicity;
- avoid speculative abstractions;
- avoid premature optimization;
- avoid introducing unnecessary infrastructure;
- avoid adding configuration unless clearly needed;
- avoid adding features "because competitors have them".

Implement the smallest valuable solution first.

Prefer:
- explicit code;
- understandable flows;
- shallow abstractions;
- incremental evolution.

Do not introduce:
- microservices;
- event-driven complexity;
- plugin systems;
- generic engines;
- unnecessary indirection.

Complexity must be earned.


---

# Technical Philosophy

Technology choices should optimize for:

- maintainability;
- development speed;
- clarity;
- low operational burden;
- self-hostability.

The project should remain understandable by a single developer.

Infrastructure should remain economically small.


---

# Performance Philosophy

The application should feel lightweight.

Key priorities:
- fast image loading;
- responsive browsing;
- smooth scrolling;
- low UI latency.

Do not sacrifice simplicity for theoretical scalability.


---

All UI generation and visual decisions must follow:

`DESIGN.md`

# Product North Star

The ideal user feeling is:

"I found something visually interesting and saved it instantly."

and later:

"I can rediscover it effortlessly."
